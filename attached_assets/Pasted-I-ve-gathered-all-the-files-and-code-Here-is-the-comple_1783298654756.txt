4. Difference: iBrowse vs iSearch
Aspect	iBrowse	iSearch
File	IneXusAgent.tsx	AgentOSAgent.tsx
State atom	inexusAgentEnabledAtom	agentOSEnabledAtom
Backend	MavericPro/api/server/routes/inexus/agent.js	MavericPro/agent-os/artifacts/api-server/src/routes/agent/messages.ts
Protocol	WebSocket (/api/inexus/ws)	SSE (/api/agent-os/agent/sessions/:id/messages)
Engine	IneXus (original agent loop in agent.js)	Agent-OS (separate API server with structured tool loop)
Max rounds	No explicit limit	8 rounds (MAX_ROUNDS = 8)
Browser control	Full (navigate, click, type, screenshot)	Full (via interact_page tool)
Search	Tavily API + page reading (top 2 pages)	Tavily API + fetch_url tool
Tool architecture	Inline function calls in agent loop	Structured tool definitions + LLM tool_choice
UI chrome	Floating/docked browser overlay	Steps sidebar + sources panel
Sender label	'iBrowse'	'iSearch'
Stop mechanism	WS send({ type: 'stop' })	sseRef.current.close()

When a skill with tool === 'search_web' matches in iBrowse mode, the backend (agent.js line 2136) calls:

const { text: searchText, results: searchResults } = await searchWeb(userContent);
